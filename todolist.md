# 🥊 快打旋風 6 (SF6) RL AI 開發紀錄 (TODO List)

本文件紀錄專案從基礎 CNN 視覺模型，升級至具備長短期記憶、精確複合輸入與防呆機制的進階多模態 RL 代理人 (Agent) 的開發歷程。

---

## ✅ 已完成項目 (Completed)

* **✅ TODO 1：解決「變長對戰」與「發呆污染」**
  * **目的**：避免對戰結束後的發呆畫面與0分獎勵污染神經網路。
  * **如何解決**：實作「環境時間凍結 (Frame Skipping)」，在 `reset()` 中凍結 Python 執行緒，直到滿血開局。
  * **可能引發的問題**：若誤入訓練房或 UI 辨識錯誤，可能引發環境無限期死鎖 (Deadlock)。

* **✅ TODO 2：升級多模態神經網路 (MultiInputPolicy)**
  * **目的**：讓 AI 精確理解自身與對手的資源狀態（血量、鬥氣百分比），學會高階資源控管。
  * **如何解決**：將觀測空間升級為 `Dict` (包含 `image` 與 `stats`)，策略更換為 `MultiInputLstmPolicy`。
  * **可能引發的問題**：狀態空間變大，初期訓練收斂時間變長。

* **✅ TODO 3：導入 LSTM 記憶網路 (局內動態對策)**
  * **目的**：賦予 AI 長短期記憶，理解動作與結果的因果關係（如揮空被確反）。
  * **如何解決**：引入 `sb3-contrib`，全面升級為 `RecurrentPPO`。
  * **可能引發的問題**：RNN 訓練速度較慢，且無法與 SB3 原生的 Off-Policy 或 MaskablePPO 混用。

* **✅ TODO 4：實作軟性動作遮罩 (Soft Action Masking)**
  * **目的**：避免 AI 在硬直或被擊飛時狂按按鍵，破壞獎勵關聯。
  * **如何解決**：在 `step()` 中實作軟性遮罩，硬直期間強制轉為待機並給予微小懲罰 (`-0.05`)。
  * **可能引發的問題**：剝奪了 AI 自己發明「招式取消 (Cancel)」的可能性，需高度依賴 JSON 巨集。

* **✅ TODO 5：強化指令執行與驗證機制 (精確複合輸入)**
  * **目的**：確保 AI 能 100% 成功放出必殺技與複合指令（如斜下）。
  * **如何解決**：升級鍵盤控制器支援 `press/release`，並加入「隨機抖動 (Jitter)」規避外掛偵測。
  * **可能引發的問題**：高負載掉幀時，寫死的 `time.sleep` 延遲可能與遊戲引擎脫鉤。

* **✅ TODO 6：獎勵函數重塑與防守機制**
  * **目的**：引導 AI 學會立回與防禦，不再當無腦狂戰士。
  * **如何解決**：當 AI 拉後防禦且鬥氣減少、血量無扣除時，給予 `+2.0` 正向獎勵。
  * **可能引發的問題**：AI 可能發現防禦穩賺不賠，演化成極度龜縮的「烏龜型 AI」。

## 🚀 進行中與待處理項目 (In Progress & Pending)

* **🚀 TODO 8：超參數調校與探索獎勵**
  * **目的**：解決 AI 因為害怕扣分或過度依賴防禦而產生的龜縮行為，鼓勵主動進攻與試錯。
  * **如何解決**：調高 PPO 的熵係數 (`ent_coef`) 強迫探索；在環境中加入微小的「主動前進/進攻」誘因獎勵。
  * **可能引發的問題**：若前進獎勵給得太高，AI 會退化成只會往前衝的沙包，破壞原本學好的防禦邏輯。

* **⏳ TODO 7：導入 TensorBoard 數據戰情室**
  * **目的**：視覺化訓練過程，監控平均勝率、傷害量、熵值等指標，不再盲目訓練。
  * **如何解決**：啟用 PPO 的 `tensorboard_log`，撰寫自訂 Callback 記錄每回合的自定義數據。
  * **可能引發的問題**：增加少許 I/O 負擔，需額外啟動伺服器查看數據。

* **⏳ TODO 9：自我對弈 / 多樣性對手池**
  * **目的**：避免 AI 對單一角色 (如 Ryu) 過度擬合 (Overfitting)，提升應對全角色的泛化能力。
  * **如何解決**：腳本定期切換遊戲內的對手，或最終實現兩個 RL Agent 互打 (Self-play)。
  * **可能引發的問題**：訓練時間指數級上升，Self-play 需要極高的硬體算力與非同步架構。

* **⏳ TODO 10：全自動化工廠 (視覺自動化與 KO 偵測)**
  * **目的**：達成 24 小時無人值守訓練，解決削血致死導致的環境死鎖，並自動點擊「再戰」。
  * **如何解決**：使用 OpenCV 模板匹配 (Template Matching) 抓取 "K.O." 字樣與 "Rematch" 按鈕，觸發自動點擊。
  * **可能引發的問題**：遊戲解析度或 UI 更新可能導致模板匹配失效；匹配閾值太低可能誤判。

* **⏳ TODO 11：戰術與強度 Meta (策略約束與確反獎勵)**
  * **目的**：賦予 AI 戰前策略切換能力（如猛攻型、龜縮型），並加速學習高階的「誘騙與確反」。
  * **如何解決**：擴充 `MultiInputPolicy` 的輸入陣列加入 `Strategy_ID`，並實作命中硬直時的「確反額外獎勵」。
  * **可能引發的問題**：狀態空間變大，學習難度驟增；人為定義的確反獎勵可能與原生勝負獎勵衝突。
